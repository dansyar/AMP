6. OPTIMIZED PROMPT - INDUSTRY DASHBOARD AND VERCEL
======================================================================

You are Codex, acting as a senior product architect, hackathon strategist, AI programme designer, and full-stack engineer.

We are building a Vercel-deployable dashboard MVP for AMP's AI Fluency ideathon.

Challenge:
"AI is reshaping every profession. AMP's next step is to build a community that does not just learn about AI, but leads with it. Design an AI Fluency ideathon that surfaces and empowers AI Champions from within the Muslim professional community to pilot, run, and scale grassroots AI learning programmes."

Target industries:
- Construction
- Architecture
- Banking
- Childcare
- Education
- Military

Core requirement:
Create industry-specific AI Fluency Blueprints for each target industry. Each blueprint should identify:
- The biggest AI knowledge gaps in that industry
- The age group or career stage with the largest AI fluency gap
- Why that group is most at risk of being left behind
- Practical AI use cases relevant to that industry
- Ethical, privacy, cultural, and workplace concerns
- What an AI Champion looks like in that industry
- A peer-led programme model for that industry
- How to measure real behaviour change, not just attendance

Product format:
Build a dashboard-style web app deployable on Vercel.

The MVP should include:
1. Industry Blueprint Dashboard
2. Industry Blueprint Detail Page
3. AI Champion Model
4. AI Fluency Measurement Model
5. AMP Admin Dashboard

Architecture requirements:
- Use Next.js + TypeScript
- Use a clean dashboard UI suitable for a professional/community organisation
- Use Tailwind CSS and preferably shadcn/ui or simple reusable components
- Use static seed data for the hackathon MVP
- Structure data cleanly so it can later move to Supabase/Postgres
- Keep business logic separate from UI components
- Include reusable types for IndustryBlueprint, AIChampionProfile, FluencyMetric, ProgrammePilot, and BehaviourChangeEvidence
- Include mock data for all six industries
- Include a production-ready Vercel deployment setup

Suggested age/career-stage logic:
For each industry, choose the age group or career stage most likely to have the biggest practical AI fluency gap. Do not assume the oldest group always has the biggest gap. Consider:
- Mid-career professionals who manage workflows but have limited AI exposure
- Senior professionals with decision-making authority but low hands-on AI confidence
- Early-career workers who use tools casually but lack ethical/workplace fluency
- Sector-specific constraints such as safety, regulation, privacy, hierarchy, or care responsibilities

For each industry, explain the reasoning behind the selected target group.

Demo flow:
1. AMP opens the dashboard and sees six industry AI Fluency Blueprints.
2. AMP selects an industry, for example Banking or Childcare.
3. The dashboard reveals the largest AI fluency gap and target age group.
4. The system shows what an AI Champion should look like in that industry.
5. The system recommends a peer-led pilot programme.
6. AMP sees how behaviour change will be measured after the session.
7. The dashboard shows how this model can scale across workplaces and community spaces.

Implementation output:
- Generate the full project architecture
- Create the Next.js app if not present
- Implement the dashboard MVP
- Add seed data for the six industries
- Add clean reusable components
- Add README with setup and deployment instructions
- Run typecheck/build
- Start a local production build where possible
- Prepare for Vercel production deployment

Vercel deployment requirement:
After implementation:
- Ensure `npm run build` passes
- Ensure the app is ready for Vercel
- If Vercel CLI is available and the project is linked, deploy to production using `vercel --prod`
- If Vercel is not configured, provide exact setup steps:
  `npm i -g vercel`
  `vercel login`
  `vercel link`
  `vercel --prod`

Be specific, pragmatic, and demo-focused. Prioritise a polished dashboard MVP over excessive backend complexity. Use synthetic/sample data only. Avoid generic AI education advice; make every blueprint industry-specific and useful for AMP's Muslim professional community.
